# Changelog

## [Unreleased] — v3 重构

### Added
- **列表视图**：训练页改为动作名称列表（不含组数据），每行显示状态圆圈（空心○/橙色⋯/绿色✓）
- **动作大卡片**：点击列表行进入全屏大卡片，专注单动作训练。布局：发力要点 → 备注框 → 组表格+Stepper → 完成/重置按钮
- **动作级备注**：大卡片内始终可见的文本框，输入即时写入 logs，归档后随历史记录展示
- **实际顺序记录**：`orderIndex` 字段记录用户完成动作的顺序，支持训练分析
- **额外动作**：列表底部"+ 添加额外动作"按钮，支持添加计划外的器械/变体，✕ 一键删除，切天自动清除
- **纯有氧日归档**：仅填写 AW 有氧数据也可归档，dayName 自动设为"有氧日"
- **进行中状态**：部分组完成的动作在列表显示橙色 ⋯ 标识，区别于未开始和已完成
- **v2→v3 自动数据迁移**：首次启动检测到 `tr_app_v2` 自动迁移到 `tr_app_v3`，完成后清除旧 key
- **已归档动作可查看备注**：历史页动作详情中显示备注行

### Changed
- 训练页交互模型：从"所有动作同时展示+滚动勾选"变为"列表导航→大卡片专注→退回选下一个"
- 存储 key：`tr_app_v2` → `tr_app_v3`
- `ExerciseLog` 新增 `note`、`orderIndex` 字段
- `HistoryRecord.details` 条目新增 `note`、`orderIndex` 字段
- 归档逻辑：遍历 `state.logs` 全部动作（含 extra- 前缀），而非仅 `DAYS` 中的
- 额外动作"靶肌肉"从下拉选择改为自由文本（不要求专业名称，填"臀""大腿""背"等即可）
- 已完成动作从"不可点击"改为可重入只读查看

### Removed
- SVG 胶囊体肌肉示意图（`svgCapsule` 函数保留为 dead code 未调用）
- `toggleExComplete` 函数（v2 状态圆圈点击逻辑，v3 不再使用）
- 训练页卡片布局中的组表格直接展示

### Fixed
- 归档时 extra 动作不写入 history 的幽灵数据问题
- v2→v3 迁移弹窗改为静默执行（单人场景）
- 已完成动作不可点击，改为可重入查看（只读模式）
